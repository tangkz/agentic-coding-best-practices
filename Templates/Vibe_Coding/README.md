# Practice 2: Vibe Coding (The Prototyping Paradigm)

## Overview
"Vibe coding" is an emerging practice where the developer describes a problem or aesthetic in natural language and allows the agent to handle the heavy lifting across the editor, terminal, and browser. It is exceptionally effective for UI generation and rapid prototyping.

## Core Mechanism
1. **High-Level Goal Identification:** Instead of micro-managing syntax, provide high-level "vibes" and goals.
2. **Autonomous Execution:** The agent proposes a checklist and moves into autonomous execution across multiple surfaces.
3. **Visual Verification:** Use browser subagents to verify UI changes in real-time.

## Efficiency Gains
- Translates abstract design concepts into functional interfaces quickly.
- Reduces the cognitive load of syntax and boilerplate.
- Achieves high task completion speeds for UI tasks.

## Template Steps

### Step 1: Design the "Vibe"
Identify the core aesthetic and functional requirements.
- Primary colors/theme: [e.g., Dark Mode, Glassmorphism]
- Core functionality: [e.g., Responsive dashboard, Real-time data visualization]
- Tech Stack: [e.g., Next.js, Tailwind CSS]

### Step 2: The Initial Prompt
Provide a comprehensive "vibe" prompt:
> "Build a responsive personal finance dashboard using Next.js and Tailwind CSS. The design should feel premium with smooth transitions and a dark mode aesthetic. Include charts for monthly spending and a list of recent transactions."

### Step 3: Visual Review
Monitor the agent's work through browser recordings and screenshots.

### Step 4: Iterative Refinement
Adjust based on visual feedback:
> "The dashboard looks great, but could you make the transaction list cards more rounded and add a subtle hover effect?"

## Best Practices
- Start with a clear mental model of the desired outcome.
- Use positive instructions (what to do, not what not to do).
- Break complex vibes into slightly smaller iterative steps if the agent starts to drift.
