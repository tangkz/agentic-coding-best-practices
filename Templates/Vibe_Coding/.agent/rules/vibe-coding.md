# Vibe Coding Rules

## Visual Standards
- Every UI change MUST be accompanied by a browser recording or screenshot.
- Prioritize responsive design (Mobile, Tablet, Desktop) in all components.

## Interaction Design
- Use smooth transitions (CSS transitions/animations) for interactive elements.
- Hover states and active states must be explicitly defined.

## Prototyping Speed
- Focus on the "Happy Path" first. Document edge cases but do not let them block the initial "vibe" liftoff.
